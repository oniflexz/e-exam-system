User registers/logs in\n   - JWT token generated\n   - Token stored in localStorage\n   - Added to all subsequent requests\n\n2. **Exam Taking**\n   - Student views available exams\n   - Clicks \"Start Exam\"\n   - Timer starts, questions loaded\n   - Answers stored in state\n   - Submit sends to backend\n   - Results calculated and stored\n\n3. **Real-time Proctoring**\n   - Socket.io connection established\n   - Activity monitored (tab switches, focus loss)\n   - Events logged to database\n   - Teacher views live dashboard\n\n## 🔒 Security Features\n\n- ✅ JWT-based authentication\n- ✅ Password hashing (bcryptjs)\n- ✅ Role-based access control\n- ✅ SQL injection prevention (parameterized queries)\n- ✅ CORS configuration\n- ✅ Input validation\n- ✅ Protected routes\n- ✅ Secure headers\n\n## 🚀 Deployment Options\n\n### Docker Compose (Recommended)\n\n```bash\ndocker-compose up -d\n```\n\n### Manual Deployment\n\n**Backend:**\n```bash\ncd backend\nnpm install\nnpm run migrate\nnpm start\n```\n\n**Frontend:**\n```bash\ncd frontend\nnpm install\nnpm run build\nnpm start\n```\n\n## 📊 API Endpoints Summary\n\n| Method | Endpoint | Description |\n|--------|----------|-------------|\n| POST | `/api/auth/register` | Register user |\n| POST | `/api/auth/login` | Login user |\n| GET | `/api/exams` | Get all exams |\n| POST | `/api/exams` | Create exam (Teacher/Admin) |\n| GET | `/api/exams/:id` | Get exam details |\n| POST | `/api/exams/:id/start` | Start exam |\n| POST | `/api/exams/:id/submit` | Submit exam |\n| GET | `/api/questions/exam/:examId` | Get exam questions |\n| GET | `/api/results/my-results` | Get student results |\n| GET | `/api/results/exam/:examId` | Get exam results (Teacher/Admin) |\n\n## 🧪 Testing\n\n**Backend Tests:**\n- Authentication controller tests\n- Authorization middleware tests\n- Database connection tests\n\n**Frontend Tests:**\n- Component rendering tests\n- User interaction tests\n- API integration tests\n\n## 📝 Code Quality\n\n- ✅ ESLint configuration\n- ✅ Code formatting standards\n- ✅ Meaningful variable names\n- ✅ Well-documented code\n- ✅ DRY principles followed\n- ✅ SOLID design patterns\n\n## 🔄 Continuous Integration\n\n**GitHub Actions Workflows:**\n\n1. **CI/CD Pipeline** -
